# CLAUDE.md

## Commands

```bash
bun dev          # Start dev server on port 3000
bun build        # Production build
bun test         # Run tests (vitest)
bun lint         # ESLint
bun check        # Prettier write + ESLint fix
```

Run a single test file: `bun vitest run src/path/to/file.test.ts`

## Package manager

Use **bun** exclusively — not npm, pnpm, or yarn.

## Architecture

**Trioxygen** is a React component library / design system showcase built with TanStack Start (SSR-capable file-based routing on top of Vite).

### Routing

- `src/routes/__root.tsx` — shell document: injects theme-init script (prevents flash), mounts Sidebar + devtools
- `src/routes/<name>.tsx` — one route per component, each exports a `Route` via `createFileRoute`
- `src/routeTree.gen.ts` — auto-generated; never edit manually (regenerated by `@tanstack/router-plugin` on `bun dev`)
- Path alias `#/*` maps to `src/*`; `@components/*` → `src/components/*`; `@/` → `src/`

### Styling

All styles use **Vanilla Extract** (`*.css.ts` files):

- `src/theme/theme.css.ts` — `vars` object (design tokens via `createGlobalTheme`), dark mode via `html.dark` class
- `src/theme/reset.css.ts` — base reset
- Each component has a co-located `<component>.css.ts` with `style()` / `styleVariants()`
- Dark mode: `__root.tsx` runs an inline script before first paint to add `class="dark"` to `<html>` from `localStorage`

### Component patterns

- Components wrap **React Aria Components** primitives (never raw HTML for interactive elements)
- Interactive state via RAC data attributes: `[data-hovered]`, `[data-pressed]`, `[data-focus-visible]`, `[data-disabled]`
- Compound component pattern: `Button.Label`, `Button.Icon` as static sub-components
- Layout primitives in `src/components/layout/`: `Stack`, `Inline`, `Container`

### Showcase system

Every route page uses shared layout components from `src/styles/Showcase.tsx`:

- `ShowcasePage` — wraps content in the sidebar layout
- `ShowcaseHero` — page title/lead
- `ShowcaseSection` — labeled section
- `ShowcaseMatrix` — renders a rows×cols grid with axis labels (useful for variant/size matrices)
- `ShowcaseGrid`, `ShowcaseRow`, `ShowcasePanel`, `ShowcaseStack`, `ShowcaseDemoCard` — layout helpers
- `CodeBlock` — syntax-highlighted code via `react-shiki`, dual theme `github-light`/`github-dark`; pass plain strings, not JSX

### Adding a new component

1. Create `src/components/<Name>/` with `<Name>.tsx`, `<name>.css.ts`, and `index.ts`
2. Create `src/routes/<name>.tsx` using `createFileRoute`
3. Add the route to the `COMPONENTS` array in `src/components/Sidebar/Sidebar.tsx`
4. Router plugin auto-updates `src/routeTree.gen.ts` on next dev start

### Typescript

1. Prefer using a top-level type-only import instead of inline type specifiers
